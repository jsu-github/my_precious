# Requirements Archive: v1.1 — Market Intelligence & Sovereign Tier System

**Archived:** 2026-04-13
**Status:** ✅ All requirements implemented (documentation tracking drift resolved)

---

## v1.1 Requirements

### Data Model Extensions

- [x] **DATA-01**: User can set and edit a `weight_per_unit` value (with `weight_unit`: g or oz) on each precious metals asset via the Edit Asset modal — implemented as `weight_per_unit` + `weight_unit` fields (supports troy-oz as well as grams, more flexible than original spec)

### Market Data Management

- [x] **MKTD-01**: User can create, edit, and delete dealers (name, optional notes) — implemented via DealerPage
- [x] **MKTD-02**: User can set "We Buy" (bid) prices per gram for multiple metals (gold bar, gold coin, silver bar, silver coin, platinum, palladium) per dealer — extended beyond gold-only in final implementation
- [x] **MKTD-03**: Ledger shows a liquidation value column computed as `quantity × toGrams(weight, unit) × getDealerRate(dealer, sub_class, product_type)` using a user-selectable dealer — implemented with unit-aware calculation

### Tier System

- [x] **TIER-01**: User can assign a Tier (0–3) to each asset via the Edit Asset modal — implemented with "Sovereign Tier" dropdown
- [x] **TIER-02**: User can add crypto holdings (BTC, XMR) as standard assets with `asset_class = 'crypto'` and manually-entered EUR current value, then assign them to Tier 3 — implemented; no auto-constraint to Tier 3 (manual convention)
- [x] **TIER-03**: User can configure target %, minimum %, and maximum % per tier on the Tier page (4 fixed tiers: 0 = Grid-Down, 1 = Digital Liquidity, 2 = The Vaults, 3 = Uncensorable Frontier) — implemented with inline inputs, blur-to-save
- [x] **TIER-04**: Dedicated Tier page shows current allocation % vs target/min/max bounds for each tier with green / amber / red status — implemented with RangeBar + StatusBadge components
- [x] **TIER-05**: Dashboard shows a Tier health summary tile with per-tier status at a glance — implemented as clickable glass-panel tile

## Out of Scope (v1.1)

- **Live/automated price feeds**: All dealer prices entered manually; automation adds integration complexity
- **Multi-metal dealer pricing (original)**: EXTENDED — gold-only was original scope; silver, platinum, palladium also implemented
- **Spread calculation (ask − bid)**: Ask price deferred; spread is meaningless with only bid
- **Rebalancing automation**: Suggestions require defining transfer/sale workflows not yet in scope
