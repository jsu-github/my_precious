# Milestone v1.1: Market Intelligence & Sovereign Tier System

**Status:** ✅ SHIPPED 2026-04-11
**Phases:** 13–16
**Total Plans:** 3 (phases 15+16 shipped without formal plans)

## Overview

Added manually-maintained dealer buy prices for gold and a 4-tier sovereign portfolio risk allocation system with configurable target/min/max bounds and visual status indicators. All features are manually driven — no live price feeds. The milestone extended the v1.0 foundation with a new dealer pricing subsystem, the tier classification system, and a dashboard health tile.

## Milestone Summary

**Key Decisions:**
- Multi-metal dealer pricing: 6 price fields per dealer (gold bar, gold coin, silver bar, silver coin, platinum, palladium) instead of gold-only per original scope
- DealerPage (full-page) instead of DealerManagementModal (modal): better UX for split-panel catalog; `DealerManagementModal.tsx` remains as dead code
- Tier health computed client-side (not server-side): DashboardPage fetches tierConfig and assets independently — no new `/api/dashboard/summary` endpoint extension (contrary to Phase 16 notes)
- `weight_per_unit` + `weight_unit` instead of `weight_per_unit_grams`: supports both grams and troy-oz, making the unit explicit

**Issues Resolved:**
- `tier = 0` falsy trap: guarded throughout with `!= null` instead of truthiness checks
- Troy-oz → gram conversion: `metalPricing.ts` utility (`toGrams()`) handles unit-aware conversion
- Phases 15+16 shipped without GSD documentation workflow: retroactive VALIDATION.md files created 2026-04-13

**Issues Deferred:**
- TIER-02 partial: no automatic constraint enforcing crypto assets default to Tier 3 — manual convention only; deferred to v1.2
- Price automation / live feeds for precious metals — deferred to v1.2

**Technical Debt Incurred:**
- `DealerManagementModal.tsx` (510 lines) is dead code — never imported anywhere; should be removed
- Phases 15+16 have no SUMMARY.md — documentation gap
- `13-VALIDATION.md` stuck at draft/nyquist_compliant: false
