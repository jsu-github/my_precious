# Requirements: v1.3 — Crypto Asset Tracking

## Milestone Goal

Extend portfolio to support individually tracked crypto assets with quantity × spot price math, gold-backed token classification (PAXG, XAUT), and detailed custody/storage tracking per asset.

---

## Requirements

### Crypto Asset Model

- [ ] **CRPT-01**: User can record a crypto holding with coin symbol (e.g. XMR, BTC, ETH) and full coin name, with quantity stored to 12 decimal places
- [ ] **CRPT-02**: User can enter and update a manual EUR spot price per coin symbol (e.g. EUR/XMR); each entry is timestamped; latest entry drives valuation
- [ ] **CRPT-03**: Dashboard and Ledger show crypto asset value calculated as qty × latest spot price for the coin symbol

### Tokenized Gold

- [ ] **CRPT-04**: User can mark a crypto asset as gold-backed (e.g. PAXG, XAUT); its value derives from qty × XAU spot price (from v1.2 spot_prices table), not a crypto spot price — 1 token = 1 troy oz

### Custody / Storage

- [ ] **CRPT-05**: User can assign a custody type to any crypto asset from: `exchange`, `hot_wallet`, `hardware_wallet`, `paper_wallet`, `custodian`
- [ ] **CRPT-06**: User can enter a free-text custody name alongside the type; UI provides per-type suggestions:
  - **exchange** → Binance, Coinbase, Kraken, Bybit, Bitfinex, Bitstamp
  - **hardware_wallet** → Ledger Nano X, Ledger Nano S, Trezor Model T, Trezor Model One, Coldcard, BitBox02
  - **hot_wallet** → MetaMask, Exodus, Electrum, Trust Wallet, Wasabi
  - **paper_wallet** → (no suggestions, free text)
  - **custodian** → (free text only)
- [ ] **CRPT-07**: Custody type and custody name are visible in the Ledger and asset detail/edit views

---

## Future Requirements (deferred)

- On-chain address tracking (blockchain explorer links)
- Automatic crypto spot price feeds
- Portfolio breakdown by custody type (exchange vs self-custody allocation view)
- Multi-wallet per asset (single custody record per asset is v1.3 scope)

---

## Out of Scope (v1.3)

- Live crypto price feeds — manual entry only, consistent with existing metals spot model
- Blockchain address validation or on-chain balance fetching
- Tax-lot crypto accounting (FIFO/LIFO for crypto disposals)
- NFT or DeFi position tracking
