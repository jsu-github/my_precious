# Requirements: v1.2 — Portfolio Valuation

**Defined:** 2026-04-13
**Core Value:** A single Global Net Worth Dashboard showing total financial position across all entities and asset classes — accurate enough to trust, fast enough to open daily.

---

## v1.2 Requirements

### Spot Price Management

- [ ] **SPOT-01**: User can manually enter the current spot price per metal (XAU / XAG / XPT / XPD) as EUR per gram
- [ ] **SPOT-02**: Spot prices are stored as an append-only history log with timestamp — the app queries the latest entry per metal on demand

### Dealer-Derived Valuation

- [ ] **DERIV-01**: User can trigger on-demand recalculation of liquidation value per asset from dealer We Buy price × weight × qty (default dealer: HollandGold); assets without dealer prices retain their manual `current_value`
- [ ] **DERIV-02**: Recalculation is user-initiated — no silent overwrites occur during normal page load

### Value / Premium Decomposition

- [ ] **PREM-01**: App shows per-acquisition value/premium breakdown for metal assets: metal value = weight × qty × spot; premium = acquisition cost − metal value; aggregated totals shown per metal sub-class (gold / silver / platinum)
- [ ] **PREM-02**: App shows premium % per acquisition (premium / acquisition_cost × 100)

### Valuation Screen

- [ ] **VAL-01**: Dedicated Valuation page shows a value summary table with three columns per metal: total cost basis | metal spot value (spot × weight) | liquidation value (dealer We Buy × weight)
- [ ] **VAL-02**: Valuation page shows a cash flow section: EUR deployed (sum of acquisition costs), EUR returned (via transfers), unrealized P&L (liquidation − cost)
- [ ] **VAL-03**: Valuation page respects the global entity filter (Personal / Business / Combined), consistent with all other pages in the app

## Future Requirements (Deferred)

- Break-even price per gram per acquisition — deferred to v1.3+
- Dry-run / preview mode before overwriting current_value — deferred to v1.3+
- Live / automated spot price feeds — out of scope (manual entry is the constraint)
- You Sell (ask) prices / spread tracking — deferred
- Price history trend charts per dealer — deferred
- IRR / time-weighted return (TWR) / money-weighted return (MWR) calculations — deferred; requires full disposal tracking
- FIFO / LIFO cost accounting — deferred
- Per-acquisition P&L over time (chart) — deferred

---

## Out of Scope (v1.2)

- **Live price feeds**: All spot and dealer prices remain manually entered — automation adds integration complexity with no architectural payoff at this stage
- **Disposal / sale tracking**: No `disposals` table — "capital returned" is approximated via the existing `transfers` table; labeled explicitly in UI
- **Crypto spot decomposition**: Crypto assets use manual `current_value` only — spot math applies to precious metals only (`asset_class = 'precious_metals'` with non-null `weight_per_unit`)
