<task type="auto">
  <name>Task 2: Create frontend/vite.config.ts + frontend/index.html</name>
  <files>frontend/vite.config.ts, frontend/index.html</files>
  <action>
Create `frontend/vite.config.ts`. The proxy target reads from `VITE_API_URL` environment variable — in docker-compose this is set to `http://api:3001` (using Docker service name), in local development outside Docker it falls back to `http://localhost:3001`:

```typescript
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';

// https://vitejs.dev/config/
export default defineConfig({
  plugins: [react()],
  server: {
    host: '0.0.0.0',
    port: 3000,
    proxy: {
      '/api': {
        // In Docker: VITE_API_URL=http://api:3001
        // Local dev: falls back to http://localhost:3001
        target: process.env.VITE_API_URL || 'http://localhost:3001',
        changeOrigin: true,
      },
    },
  },
});
```

<task type="auto">
  <name>Task 3: Create frontend/Dockerfile</name>
  <files>frontend/Dockerfile</files>
  <action>
Create `frontend/Dockerfile` for development mode (hot-reload). Source code is volume-mounted from host at runtime via docker-compose, so only node_modules are baked into the image:

```dockerfile
FROM node:20-alpine
WORKDIR /app

# Install dependencies into the container image layer
# (source files are volume-mounted from host at runtime)
COPY package*.json ./
RUN npm ci

# Expose Vite dev server port
EXPOSE 3000

# Run Vite dev server with hot-reload (source mounted via docker-compose volumes)
CMD ["npm", "run", "dev"]
```
